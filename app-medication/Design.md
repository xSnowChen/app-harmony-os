本次仅修改 **app-medication 应用的单一主页面**（原有用药提醒页面），不改动其他页面，严格按本修订版规范重构，重点调整页面布局顺序（提醒卡片置顶，拍照按钮置下），方便老人操作。

核心原则：大字、高对比、少元素、无冗余，适配老人使用，与用药提醒功能严格匹配，所有尺寸、颜色延续原有规范，仅调整布局顺序。

## 一、整体页面规范（核心：单一页面，无跳转，布局顺序调整）

- 页面布局：上下结构，**最顶部为「今日提醒」提示文字+提醒卡片列表**，中间为拍照按钮，**最底部为原有公共按钮**（返回应用中心、打开监控中心），无其他页面跳转；核心约束：公共按钮必须固定在页面最底部，始终可见，无论提醒卡片多少、是否滑动，均不被遮挡，确保老人随时能找到公共按钮

- 页面背景：纯浅灰色 #F5F5F5，无任何渐变、纹理、装饰性图案

- 边距规范：页面左右边距 32px，上下边距 24px，元素之间间距 24px（避免拥挤，方便老人视觉区分）

- 无多余元素：不添加任何装饰图标、分割线（除核心内容外），禁止广告、水印，仅保留指定提示文字


## 二、核心元素设计规范（重点调整布局顺序）

### 2.1 今日提醒提示文字（顶部，优先展示）

- 位置：页面最顶部，提醒卡片列表**左侧**，与页面顶部间距 24px

- 字体：鸿蒙默认字体，34px，黑色 #333333，加粗，无斜体、无下划线

- 样式：仅纯文字”今日提醒”，无背景、无边框、无装饰，与第一张提醒卡片间距 10px（贴合要求，不太远）


### 2.2 提醒卡片（核心展示，位于今日提醒下方）

卡片样式：圆角 24px，白色背景 #FFFFFF，无阴影、无边框，单张卡片高度 180px（适配新增内容），宽度与页面适配（左右留边后全屏）

卡片内部布局（严格按要求排列，清晰有序）：

- 提醒时间：左上位置，字体 35px，黑色 #333333，加粗，无多余修饰

- 提醒状态：右上位置，与提醒时间水平对齐，字体 32px，未提醒=绿色 #34A853，已提醒=灰色 #9E9E9E，加粗

- 药品图片：左下位置，尺寸 80px×80px，居中裁剪，无圆角、无边框、无滤镜，与提醒时间垂直对齐
    
- 药品名称：右下位置，与药品图片水平对齐，字体 34px，黑色 #333333，无修饰，与药品图片间距 10px（不要太远）
    
- 服用剂量：药品名称正下方，字体 25px，颜色 #757575（浅灰色），无修饰，与药品图片间距 10px（不要太远）
    
- 底部提示文字（新增，属于卡片内部元素）：卡片最底部，居中对齐，字体 14px，颜色 #9E9E9E（灰色），纯文字“长按可删除此提醒”，无背景、无边框，确保不出现换行，属于每个提醒卡片的固有内容，每张卡片均需显示（缩小后确保能正常显示，解决不显示问题）
    

卡片间距：单张卡片占满宽度，多张卡片上下排列，间距 24px，无横向排列（避免老人左右滑动）；有多条提醒时，支持上下滑动查看，滑动无卡顿、无动画，滑动区域为提醒卡片列表整体，适配老人操作。

无提醒卡片时展示：当无任何用药提醒时，显示提示文本“今日还没有提醒”，该文本与“今日提醒”提示文字间距 20px，居中显示，字体 40px，黑色 #333333，无背景、无边框、无装饰，与页面整体风格一致。

### 2.3 拍照按钮（核心操作，位于提醒卡片下方）

- 尺寸：直径 120px（固定），圆角 60px（全圆角，无棱角）

- 颜色：主色 #FF6B6B（珊瑚红，无渐变、无阴影），点击时变色为 #E55353（深一点的珊瑚红），松开恢复原色

- 位置：提醒卡片列表下方，居中放置，与上方提醒卡片列表间距 24px，与下方公共按钮间距根据底部预留空间自适应

- 交互反馈：点击时震动 50ms + 颜色变化，无其他弹窗、动画

- 位置：页面最底部，固定布局（不随页面滑动而移动），居中/按原有排列方式放置，与上方提醒卡片列表间距 48px；核心要求：公共按钮始终悬浮/固定在底部，不被提醒卡片、滑动区域遮挡，确保滑动到任何位置都能看见

#### 操作流程
```
点击拍照按钮 → 选择药品图片 → 语音询问"请说药品名称"
→ 用户说药品名 → 语音询问"一天吃几次？"

→ 用户说剂量 → 语音确认设置内容 → 自动创建提醒
```

#### 语音交互规范
- **药品名称**：用户说的内容直接作为药品名称
- **频率解析**：
  - "一天一次" / "一次" → 08:00
  - "一天两次" / "两次" → 08:00, 17:00

```typescript
  const chineseToNumber: Record<string, string> = {
    '半': '0.5', '一': '1', '二': '2', '两': '2', '三': '3',
    '四': '4', '五': '5', '六': '6', '七': '7', '八': '8', '九': '9', '十': '10'

    return digitMatch[1] + digitMatch[2];
  }

  // 2. 匹配 "中文数字+单位" 格式，转换为阿拉伯数字
  const chineseNums = ['半', '一', '二', '两', '三', '四', '五', '六', '七', '八', '九', '十'];

      }
    }
  }
```

### 2026-04-03 更新：语音文本同步彻底修复

#### 问题根因分析

用户反馈："语音重复了复方丹参片，还没说完，文本就显示一天吃几次"

**底层问题**（不是参数问题，是架构缺陷）：

| 问题层 | 具体缺陷 | 影响 |
|-------|---------|-----|
| TTS引擎层 | `onComplete`事件可能丢失 | Promise永远不resolve |
| 时序漏洞 | listener设置和speak调用竞态 | 事件触发时listener未就绪 |
| 无保底机制 | 无超时兜底 | 卡住时代码阻塞或跳过 |

#### 修复方案

**1. VoiceService.speak()重构**
```typescript
async speak(text: string): Promise<void> {
  // 关键改进：
  // 1. 超时保底：按文本长度计算，每字300ms，最少3秒
  // 2. 防多次resolve：resolved标志位
  // 3. requestId校验：确保只响应当前请求
  // 4. 播报后缓冲：onComplete后额外等待300ms
  // 5. TTS不可用兜底：等待1.5秒让文本显示停留

  const requestId = Date.now().toString();
  let resolved = false;
  const estimatedDuration = Math.max(3000, text.length * 300 + 500);

  timeoutId = setTimeout(() => {
    if (!resolved) { resolved = true; resolve(); }
  }, estimatedDuration);

  // 先设置listener，再调用speak（修复时序漏洞）
  this.ttsEngine!.setListener({
    onComplete: (reqId, response) => {
      if (!resolved && reqId === requestId) {
        resolved = true;
        clearTimeout(timeoutId);
        // 关键：额外300ms缓冲，确保声音完全结束
        setTimeout(resolve, 300);
      }
    }
  });

  this.ttsEngine!.speak(text, { requestId, extraParams: {} });
}
```

**2. Index.ets确认阶段等待时间延长**
```typescript

private static getInitialsFromText(text: string): string {
  const initials: string[] = [];
  for (const char of text) {
    const initial = MedicineDatabase.charToInitial.get(char);
    if (initial) initials.push(initial);
```
